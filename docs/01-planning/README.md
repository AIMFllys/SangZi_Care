# 📋 项目规划文档

> **用途**：项目整体规划、需求分析、产品设计  
> **受众**：项目经理、产品经理、开发团队

---

## 📄 文档列表

### PRODUCT_SPEC.md（产品规格书）⭐
**最重要的文档**，包含：
- 核心交互模型（语音助手、手势操作）
- 家属绑定系统
- 所有功能模块详细规格（用药管家、健康记录、捂话功能等）
- 技术实现要点
- 已确认的技术决策

**适合阅读对象**：
- 新加入的开发者（必读）
- 产品经理
- UI/UX 设计师

---

### plan.md（开发计划）
**开发路线图**，包含：
- 分阶段开发计划（Phase 1-3）
- 每个阶段的功能清单
- 时间估算
- 优先级排序

**适合阅读对象**：
- 项目经理
- 开发团队负责人
- 需要了解项目进度的人

---

### PROJECT_DEEP_ANALYSIS.md（项目深度分析）
**技术方案分析**，包含：
- 技术栈选型理由
- 架构设计思路
- 可用的 Skills 和 Agents
- Kiro 配置建议

**适合阅读对象**：
- 技术负责人
- 架构师
- 需要了解技术细节的开发者

---

## 🔄 文档更新规则

### 何时更新这些文档？

1. **PRODUCT_SPEC.md**
   - 功能需求变更时
   - 交互设计调整时
   - 技术决策变更时
   - 添加新功能时

2. **plan.md**
   - 开发计划调整时
   - 阶段完成时
   - 优先级变更时

3. **PROJECT_DEEP_ANALYSIS.md**
   - 技术栈变更时
   - 架构调整时
   - 添加新的技术方案时

### 更新流程

1. 修改文档
2. 更新文档顶部的"更新日期"
3. 在文档底部添加更新记录（可选）
4. 提交 Git commit
5. 通知相关团队成员

---

## 📌 重要提示

- 这些文档是项目的"单一真相来源"（Single Source of Truth）
- 所有功能开发都应基于这些文档
- 如有疑问或冲突，以 `PRODUCT_SPEC.md` 为准
- 定期（每周）检查文档是否与实际开发同步
