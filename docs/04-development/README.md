# 📝 开发记录

> **用途**：开发日志、会议记录、问题追踪、调研报告  
> **受众**：开发团队、项目管理

---

## 📂 目录结构

### report_record/（调研报告）
**项目前期调研资料**，包含：
- 采访提纲
- 调研日志
- 音频记录
- 照片视频
- 调研结果报告

**说明**：
- 这是项目启动前的调研资料
- 主要用于了解用户需求和痛点
- 已归档，不再频繁更新

---

## 📝 待创建的文档

### CHANGELOG.md（变更日志）⏳
**记录所有重要变更**，应包含：
- 版本号
- 发布日期
- 新增功能
- Bug 修复
- 破坏性变更

**格式示例**：
```markdown
## [1.0.0] - 2026-03-01

### Added
- 语音助手功能
- 用药管家功能
- 健康记录功能

### Fixed
- 修复语音识别延迟问题
- 修复数据库连接超时

### Changed
- 优化 UI 交互流程
- 更新数据库索引
```

---

### MEETING_NOTES.md（会议记录）⏳
**团队会议记录**，应包含：
- 会议日期和参与者
- 讨论议题
- 决策结果
- 待办事项

**格式示例**：
```markdown
## 2026-02-25 技术评审会议

**参与者**：张三、李四、王五

**议题**：
1. 数据库设计评审
2. API 接口设计
3. 前端组件规划

**决策**：
- ✅ 通过数据库设计方案
- ⏳ API 接口需要补充文档
- 🔄 前端组件下周再讨论

**待办事项**：
- [ ] 张三：完成 API 文档
- [ ] 李四：创建前端组件库
```

---

### ISSUES.md（问题追踪）⏳
**已知问题和解决方案**，应包含：
- 问题描述
- 复现步骤
- 解决方案
- 状态（待解决/已解决）

**格式示例**：
```markdown
## 问题列表

### #001 - Supabase 连接超时 ✅ 已解决
**描述**：偶尔出现数据库连接超时
**复现**：高并发查询时
**解决方案**：增加连接池大小
**解决日期**：2026-02-25

### #002 - 语音识别延迟 🔄 进行中
**描述**：语音识别响应时间过长
**复现**：网络较慢时
**解决方案**：待定
```

---

### TECH_DEBT.md（技术债务）⏳
**需要优化的技术问题**，应包含：
- 问题描述
- 影响范围
- 优先级
- 计划解决时间

**格式示例**：
```markdown
## 技术债务清单

### 高优先级 🔴
- [ ] 数据库查询性能优化
- [ ] API 响应时间优化

### 中优先级 🟡
- [ ] 代码重构：用户模块
- [ ] 添加单元测试

### 低优先级 🟢
- [ ] 更新依赖版本
- [ ] 优化打包体积
```

---

### PERFORMANCE.md（性能记录）⏳
**性能测试和优化记录**，应包含：
- 测试日期
- 测试场景
- 性能指标
- 优化措施
- 优化效果

## 🔄 文档维护

### 日常记录

- **每日**：更新开发日志（可选）
- **每周**：更新会议记录
- **每次发布**：更新 CHANGELOG
- **发现问题时**：更新 ISSUES

### 定期回顾

- **每周**：回顾待办事项
- **每月**：回顾技术债务
- **每季度**：回顾性能指标

---

## 💡 使用建议

### 记录开发过程

- 遇到问题时记录到 ISSUES.md
- 做出重要决策时记录到 MEETING_NOTES.md
- 发现技术债务时记录到 TECH_DEBT.md
- 发布新版本时更新 CHANGELOG.md

### 查找历史信息

- 想了解某个功能何时添加 → 查看 CHANGELOG.md
- 想了解某个问题如何解决 → 查看 ISSUES.md
- 想了解某个决策的背景 → 查看 MEETING_NOTES.md
