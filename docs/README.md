## �📝 文档编写规范

### 文件命名规范

```
核心文档：PRODUCT_SPEC.md（全大写 + 下划线）
普通文档：quick-start.md（小写 + 连字符）
归档文档：可使用中文命名
```

### 文档结构模板

```markdown
# 📄 文档标题

> **简短描述**  
> **创建日期：** YYYY-MM-DD  
> **更新日期：** YYYY-MM-DD

---

## 正文内容

---

## 相关文档

- [相关文档1](link)
- [相关文档2](link)

---

*文档说明 · 桑梓智护项目*
```

### 文档更新规则

| 更新类型 | 操作 |
|---------|------|
| 重大更新 | 更新顶部"更新日期" + Git commit |
| 小修改 | 文档底部添加更新记录 |
| 废弃文档 | 移动到 `04-development/archived/` |
| 新增文档 | 更新对应目录的 README.md |

## � 贡献指南

### 如何贡献文档？

1. **发现问题** - 文档错误、缺失、过时
2. **创建分支** - `git checkout -b docs/update-xxx`
3. **编写/修改** - 遵循文档规范
4. **提交审核** - 创建 Pull Request
5. **合并发布** - Review 通过后合并

### 文档质量标准

- ✅ 内容准确、完整
- ✅ 格式统一、规范
- ✅ 示例清晰、可运行
- ✅ 链接有效、正确
- ✅ 语言简洁、易懂
