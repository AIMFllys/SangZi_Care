## 📝 待创建的文档

### HEALTH_BROADCAST_PROMPTS.md（健康广播提示词）⏳
**健康广播 AI 生成提示词**，应包含：
- 基础 System Prompt
- 不同类别的提示词模板（营养、运动、季节、用药、睡眠）
- 用户画像参数
- 输出格式要求
- 示例对话

**格式示例**：
```markdown
## 营养类健康广播

### System Prompt
你是一位资深营养师和中医养生专家，名叫"小护"。

请根据以下信息，生成一段2-3分钟的健康科普内容：

用户信息：
- 年龄：{age}岁
- 慢性病：{diseases}
- 当前季节：{season}
- 今日天气：{weather}

要求：
1. 语言通俗易懂，像跟老人聊天
2. 包含1-2个实用饮食建议
3. 避免专业医学术语
4. 不做诊断，不推荐具体药物
5. 语气温暖亲切
6. 结尾总结一句话"记住"

### 示例输出
"张爷爷，今天给您聊聊冬天怎么吃得暖和又健康..."
```

### VOICE_ASSISTANT_INTENTS.md（语音助手意图识别）⏳
**语音助手意图识别规则**，应包含：
- 所有支持的意图类型
- 每个意图的触发关键词
- 实体提取规则
- 意图优先级
- 示例对话

**格式示例**：
```markdown
## 意图列表

### record_health（记录健康数据）
**触发关键词**：
- "记一下血压"
- "我的血压是"
- "帮我记录"

**实体提取**：
- 血压值：正则匹配 "高压(\d+)低压(\d+)"
- 血糖值：正则匹配 "血糖(\d+\.?\d*)"

**执行动作**：
- 调用 health_records 表插入数据
- 返回确认信息

**示例对话**：
用户："帮我记一下血压，高压135低压88"
AI："好的，已经帮您记录了血压：135/88 mmHg"
```

### MEDICATION_REMINDER_PROMPTS.md（用药提醒提示词）⏳
**用药提醒语音播报提示词**，应包含：
- 提醒时的语音内容模板
- 不同时段的问候语
- 延迟提醒的话术
- 家属通知的话术

**格式示例**：
```markdown
## 用药提醒语音

### 正常提醒
"张爷爷，{时段}好！该吃{药品名称}了，{剂量}。"

### 延迟提醒（第1次）
"张爷爷，您还没吃{药品名称}哦，记得按时吃药。"

### 延迟提醒（第2次）
"张爷爷，{药品名称}已经延迟{时长}了，请尽快服用。"

### 家属通知
"您好，张爷爷的{药品名称}已延迟{时长}未服用，请关注。"
```

### WUHUA_MESSAGE_PROMPTS.md（捂话功能提示词）⏳
**AI 捂话生成提示词**，应包含：
- 用户意图识别
- 内容提取规则
- 语音生成提示词
- 情感表达规则

**格式示例**：
```markdown
## AI 捂话生成

### 用户说："给我女儿捂个话，让她周末来看我"

### 意图识别
- 动作：发送捂话
- 接收者：女儿
- 内容：周末来看我

### 生成内容
文字版："小红，周末有空来看看我吧。"
语音版：使用温暖、期待的语气

### System Prompt
你是一位老年人，想给家人发送语音留言。
请将用户的意图转换为自然、温暖的语言。

要求：
1. 保持老年人的说话风格
2. 语气温和、亲切
3. 简洁明了
4. 表达真实情感
```

## 📊 提示词管理

### 提示词版本控制

- 所有提示词都应纳入 Git 版本控制
- 重大变更需要记录原因
- 定期评估提示词效果

### 提示词测试

- 每个提示词都应有测试用例
- 记录测试结果和效果
- 持续优化提示词

- 监控效果

- 格式要求
- 禁止事项
